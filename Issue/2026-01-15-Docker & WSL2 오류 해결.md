# 🚀 TIL: Docker & WSL2 오류 해결 기록

## ❌ 발생한 에러
- **메시지**: `request returned 500 Internal Server Error... check if the server supports the requested API version`
- **원인**: Docker Desktop의 엔진인 WSL2 커널이 설치되지 않았거나 업데이트가 필요함.

## 🛠️ 해결 과정
1. **상태 확인**: 터미널에서 `wsl --status` 입력.
2. **원인 파악**: "WSL 2 커널 파일을 찾을 수 없습니다"라는 메시지 확인.
3. **조치**: 
   - 터미널에 `wsl --update` 실행.
   - Docker Desktop을 완전히 종료 후 재시작.
4. **결과 확인**: `docker run hello-world`가 정상 작동함.

## 💡 배운 점
- Docker는 윈도우에서 리눅스 환경(WSL2)을 빌려 쓰기 때문에, WSL2가 최신 상태여야 엔진이 정상적으로 돌아간다.
- 에러 메시지에 있는 `Internal Server Error`는 주로 통신 대상(Docker Engine)이 꺼져 있을 때 발생한다.
